## 什么是triton
Triton是一种专为GPU编程设计的高级编程语言。它提供了一个高效且易于使用的接口，使开发者能够更轻松地编写高性能的GPU代码。Triton的设计目标是简化GPU编程的复杂性，同时提供强大的功能和灵活性，使开发者能够充分利用GPU的计算能力。
triton 的核心理念是基于分块的编程范式可以促进神经网络的高性能计算核心的构建,CUDA 编写属于传统的 “单程序，多数据” GPU 执行模型，在线程的细粒度上进行编程，Triton 是在分块的细粒度上进行编程。

## triton的语法
1. **内核函数**：使用`@triton.jit`装饰器定义内核函数，内核函数是GPU上执行的代码块。
2. **指针参数**：内核函数的参数通常是指向GPU内存的指针，表示输入和输出数据。
3. **编译时常量**：使用`tl.constexpr`定义编译时常量，triton要求所有的张量维度必须是编译时常量，这样编译器才能进行优化。
4. **内置函数**：triton提供了一系列内置函数，如`tl.arange`、`tl.load`和`tl.store`，用于生成范围内的整数序列、从GPU内存加载数据和将数据存储到GPU内存中。
   1. `tl.arange(start, end)`：生成一个从`start`到`end-1`的整数序列。
   2. `tl.load(ptr, mask=None, other=0.0)`：从GPU内存中加载数据，`ptr`是指向内存的指针，`mask`是一个布尔数组，指示哪些元素应该被加载，`other`是一个可选参数，指定在`mask`为False时返回的值。例如：
   ```python
   x = tl.load(x_ptr + global_off_x, mask=mask, other=0.0)
   ```
   3. `tl.store(ptr, value, mask=None)`：将数据存储到GPU内存中，`ptr`是指向内存的指针，`value`是要存储的值，`mask`是一个布尔数组，指示哪些元素应该被存储。例如：
   ```python
   tl.store(z_ptr + global_off_x, x + 10.0, mask=mask)
   ```
5. **程序ID**：使用`tl.program_id(dim)`获取当前线程块在指定维度上的ID，这对于计算全局偏移非常有用。例如：
```python
global_off_x = off_x + tl.program_id(0) * B0
```

## 并行度的设置
在triton中，并行度是通过定义线程块的大小和数量来设置的。线程块的大小通常由编译时常量定义，而线程块的数量则由程序ID决定。通过合理地设置线程块的大小和数量，可以充分利用GPU的计算资源，提高程序的性能。

### 加法实现
向量加法：
```python
@triton.jit
def add_mask2_kernel(x_ptr, z_ptr, N0, B0: tl.constexpr):
    # 1. 算出当前 Block 的局部偏移
    off_x = tl.arange(0, B0)
    
    # 2. 计算全局偏移 (局部偏移 + Block起始位置)
    global_off_x = off_x + tl.program_id(0) * B0
    
    # 3. 使用【全局偏移】来计算 mask！
    mask = global_off_x < N0
    
    # 4. 在 load 和 store 中都使用全局偏移
    x = tl.load(x_ptr + global_off_x, mask=mask, other=0.0)
    tl.store(z_ptr + global_off_x, x + 10.0, mask=mask)
    
    return
```
B0代表线程块的大小，N0代表向量的长度。通过计算全局偏移和使用掩码，可以确保线程只处理有效的数据范围。
